
# tree-sitter-gdscript

[简体中文](README.zh-CN.md)

Improved fork of [prestonknopp/tree-sitter-gdscript](https://github.com/prestonknopp/tree-sitter-gdscript):

- Reorganized as a Neovim plugin for local parser installation via nvim-treesitter
- Removed unused language bindings
- Fixed inner class region definitions interfering with fold parsing
- Added foldable regions (`#region` / `#endregion`)
- Fixed `not` / `!` operator precedence

## Installation

Requires [nvim-treesitter](https://github.com/nvim-treesitter/nvim-treesitter), which handles parser compilation.

### lazy.nvim
```lua
{
  "asunbb/tree-sitter-gdscript",
    dependencies = {
      "nvim-treesitter/nvim-treesitter"
    }
}
```

After installing, run:
```bash
:TSUpdate
:TSInstall gdscript_n
```
nvim-treesitter will compile the parser and deploy query files automatically.

## Local Development

```bash
# Install dependencies
npm install

# After modifying grammar.js, regenerate and test
npm run genTest

# After modifying only src/scanner.c, test directly
npm run test

# Format
npm run format
```

## How It Works

The plugin injects local parser info into nvim-treesitter's parser registry via a `User TSUpdate` autocommand, including the grammar source path and query file directory.

Compilation and installation are handled by nvim-treesitter's standard flow.

## Notes

The initial scm files originate from the nvim-treesitter repository and may be improved over time.

If the gdscript parser provided by nvim-treesitter is also installed, it will conflict with this local plugin due to the same project name.
