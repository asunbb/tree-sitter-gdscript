# 外部扫描器 (scanner.c)

scanner.c 是手写 C 代码（629 行），处理 grammar.js 的 `externals` 中声明的 token。这些 token 涉及上下文状态（缩进栈、字符串分隔符栈），无法用纯 grammar DSL 表达。

tree-sitter 的外部扫描器协议要求实现 5 个函数：

| 函数 | 作用 |
|------|------|
| `create` | 分配并初始化 Scanner 结构 |
| `destroy` | 释放内存 |
| `serialize` | 将状态序列化到缓冲区（tree-sitter 增量解析需要） |
| `deserialize` | 从缓冲区恢复状态 |
| `scan` | 核心扫描函数 |

---

## 数据结构

### TokenType 枚举

```c
enum TokenType {
    NEWLINE,            // 换行符
    INDENT,             // 缩进增加
    DEDENT,             // 缩进减少
    STRING_START,       // 字符串开始引号
    STRING_CONTENT,     // 字符串内容
    STRING_END,         // 字符串结束引号
    STRING_NAME_START,  // 字符串名开始（&"）
    NODE_PATH_START,    // 节点路径开始（^" / @"）
    CLOSE_PAREN,        // 右括号 )
    CLOSE_BRACKET,      // 右方括号 ]
    CLOSE_BRACE,        // 右花括号 }
    COMMA,              // 逗号
    BODY_END,           // 复合语句体结束
};
```

### Flags 位标志

```c
typedef enum {
    SingleQuote = 1 << 0,   // 单引号
    DoubleQuote = 1 << 1,   // 双引号
    Triple       = 1 << 2,   // 三引号
    Raw          = 1 << 3,   // r"" 原始字符串
    Name         = 1 << 4,   // &"" 字符串名
    NodePath     = 1 << 5,   // ^"" / @"" 节点路径
} Flags;
```

### Delimiter 结构

```c
typedef struct {
    char flags;   // Flags 位组合
} Delimiter;
```

跟踪当前打开的字符串分隔符类型。

### Scanner 结构

```c
typedef struct {
    indent_vec *indents;     // 缩进栈（uint16_t 动态数组）
    delimiter_vec *delimiters; // 字符串分隔符栈（Delimiter 动态数组）
} Scanner;
```

两个栈分别维护缩进层级和嵌套字符串状态。

### 动态数组宏

```c
VEC_PUSH(vec, el)    // 追加元素，自动扩容
VEC_POP(vec)         // 弹出最后一个元素（len--）
VEC_BACK(vec)        // 访问最后一个元素
VEC_NEW              // 初始化空向量 {0, 0, NULL}
VEC_FREE(vec)        // 释放内存
VEC_CLEAR(vec)       // 清空（len = 0）
VEC_GROW(vec, cap)   // 扩容到指定容量
VEC_RESIZE(vec, cap) // 重新分配内存
```

## 序列化/反序列化

tree-sitter 可以在解析中途暂停并恢复（增量解析），扫描器必须能序列化/反序列化其状态。

### 序列化格式

```
[delimiter_count: 1字节]
[delimiter_0 flags: 1字节] [delimiter_1 flags: 1字节] ...
[indent_1: 1字节] [indent_2: 1字节] ...
```

- 第一个字节是分隔符数量（最大 255）
- 然后是每个分隔符的 flags
- 最后是缩进栈（从 index 1 开始，因为 index 0 始终是 0）

### 反序列化

1. 清空两个栈
2. 压入基础缩进 0（`VEC_PUSH(indents, 0)`）
3. 从缓冲区恢复分隔符栈和缩进栈

## scan 函数详解

`tree_sitter_gdscript_external_scanner_scan` 是核心函数，按阶段执行：

### 前置检测

```c
bool error_recovery_mode = valid_symbols[STRING_CONTENT] && valid_symbols[INDENT];
bool within_brackets = valid_symbols[CLOSE_BRACE] || valid_symbols[CLOSE_PAREN] || valid_symbols[CLOSE_BRACKET];
```

- `error_recovery_mode`：STRING_CONTENT 和 INDENT 同时有效说明解析器在错误恢复中
- `within_brackets`：当前是否在括号内

### 阶段 1：字符串内容解析

**条件**：`STRING_CONTENT` 有效 且 分隔符栈非空 且 非错误恢复模式

```
循环处理字符：
  遇到 '\' (反斜杠):
    原始字符串 → 跳过反斜杠和转义的引号/反斜杠
    普通字符串 → 标记当前位置为 STRING_CONTENT 结束，返回 has_content

  遇到结束引号:
    三引号 → 检查连续三个引号：
      三个引号且有内容 → STRING_CONTENT
      三个引号且无内容 → STRING_END，弹出分隔符
      不足三个 → STRING_CONTENT

    单引号:
      有内容 → STRING_CONTENT
      无内容 → STRING_END，弹出分隔符

  其他字符 → advance，has_content = true
```

### 阶段 2：空白符跳过和缩进跟踪

```c
uint32_t indent_length = 0;
bool found_end_of_line = false;
uint32_t last_non_empty_indent = 0;
```

`skip_whitespace` 处理：

| 字符 | 行为 |
|------|------|
| `\n` | indent_length 重置为 0，设 found_end_of_line = true |
| 空格 | indent_length++ |
| `\t` | indent_length += 8（制表符 = 8 列） |
| `\r` `\f` | indent_length 重置为 0 |

`last_non_empty_indent` 跟踪最后一个包含实际内容（注释或代码）的行的缩进级别，防止空行导致过早 DEDENT。

### 阶段 3：注释处理

当扫描到 `#` 字符：

1. 如果还没遇到过换行（`!found_end_of_line`），直接 break——注释在当前行，让 grammar 的 `comment` 规则处理
2. 否则更新 `last_non_empty_indent`

**Region 标记检测**（自定义新增）：

```c
if (comment_indent_length == 0) {
    // 检查 #region 或 #endregion
    is_region_marker = lookahead_string(lexer, "region" / "endregion");
}
```

Column 0 的 `#region`/`#endregion` 标记不调整缩进。

**去缩进注释处理**：

```
func example():
    # 缩进的注释
# 这里缩进为 0，但应属于函数体
    var x = 10
```

扫描器会向前看下一行内容：如果下一行有缩进，说明这个 column 0 注释属于函数体，将其 indent_length 调整为函数级缩进。

### 阶段 4：行续行符

```
遇到 '\' → 跳过
  如果后面是 '\r' → 跳过
  如果后面是 '\n' 或 EOF → 跳过（行续行成功）
  否则 → return false（非法续行）
```

### 阶段 5：EOF 处理

```
如果 last_non_empty_indent > 0:
    indent_length = last_non_empty_indent  // 使用最后有内容行的缩进
检查缩进栈 → 发出最终的 DEDENT
found_end_of_line = true
```

防止文件末尾空行导致丢失最后的 DEDENT。

### 阶段 6：内容行检测

```
如果 indent_length == 0 且 last_non_empty_indent > 0:
    // 到达 column 0 内容，但之前有更高缩进的内容
    如果 last_non_empty_indent == 栈顶缩进:
        return false  // 推迟 DEDENT
```

### 阶段 7：INDENT/DEDENT/NEWLINE 发射

```
如果 found_end_of_line:
    如果 indent_length > 栈顶 且 INDENT 有效:
        压入栈，返回 INDENT
    如果 indent_length < 栈顶 且 DEDENT 有效:
        弹出栈，返回 DEDENT
    如果 NEWLINE 有效 且 非错误恢复:
        返回 NEWLINE
```

优先级：INDENT/DEDENT > NEWLINE。

### 阶段 8：BODY_END 发射

```
条件（全部满足）：
  !valid_symbols[COMMA]
  !valid_symbols[CLOSE_PAREN]
  !valid_symbols[CLOSE_BRACE]
  !valid_symbols[CLOSE_BRACKET]
  (error_recovery_mode || valid_symbols[BODY_END])

如果 lookahead 是 ',' / ')' / '}' / ']':
    如果 DEDENT 有效 → 弹出缩进栈
    返回 BODY_END
```

BODY_END 表示"缩进块在遇到闭合括号前结束"，不消耗该括号字符。这让 grammar 可以在括号内正确结束函数体。

**为什么需要 BODY_END**：

```gdscript
func foo(x):
    return x
func bar(): pass
```

没有分号或空行分隔时，`func bar` 的 `func` 关键字在 `foo` 函数体之后。但：

```gdscript
var arr = [func(x):
    return x]
```

函数体在 `]` 前必须结束——BODY_END 解决了这个问题。

### 阶段 9：字符串开始检测

```
如果 STRING_START / STRING_NAME_START / NODE_PATH_START 有效:
    创建新 Delimiter

    检查前缀字符：
      'r' → set_raw, advance
      '&' → set_name, advance
      '^' → set_node_path, advance
      '@' → set_node_path, advance (兼容 3.x)

    检查引号字符 (' 或 ")：
      handle_quote → 设置结束字符，检测三引号

    压入分隔符栈
    返回 STRING_START / STRING_NAME_START / NODE_PATH_START
```

## 括号跟踪机制

CLOSE_PAREN / CLOSE_BRACKET / CLOSE_BRACE 作为 external 存在，**不是为了被扫描器返回**，而是让扫描器知道当前是否在括号内。

在括号内时，扫描器不发出 INDENT/DEDENT，避免：

```gdscript
var arr = [
    1,
    2,   # 这里的换行不应触发 DEDENT
    3
]
```

如果 `]` 有效（即解析器期望 `]`），扫描器处于 `within_brackets` 状态，跳过缩进处理。

## lookahead_string 辅助函数

```c
static bool lookahead_string(TSLexer *lexer, const char *string) {
    const char *cursor = string;
    while (*cursor && lexer->lookahead == *cursor) {
        skip(lexer);  // 不消费（skip = advance as extra）
        cursor++;
    }
    return (*cursor == '\0');
}
```

用 `skip`（不消费 token）逐字符匹配字符串，匹配完后 lexer 位置不变（因为 skip 的字符不计入 token 范围）。用于检测 `#region` 和 `#endregion` 标记。
