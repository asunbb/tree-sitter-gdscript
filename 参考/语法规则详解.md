# 语法规则详解 (grammar.js)

本文档逐条解析 grammar.js 中的规则定义。

---

## 优先级表 (PREC)

grammar.js 开头定义了 22 个优先级，数值越高绑定越紧：

```
-1  typed_parameter   参数类型标注（最低，避免与 for..in 冲突）
-1  conditional       三元条件表达式（右结合，低优先级）
 1  parenthesized_expression  括号表达式
 3  or                or / ||
 4  and               and / &&
 5  boolean_not       not / !  ← 自定义修复：原先与 unary 同级，现在低于比较
 6  in                in / not in
 7  compare           < <= == != >= >
 8  bitwise_or        |
 9  bitwise_and       &
10  xor               ^
11  shift             << >>
12  plus              + -
13  times             * / ** %
14  power             （未单独使用，** 在 times 中）
15  unary              - + ~ （一元运算符）
16  is                 is / is not
17  as                 as
18  call              函数调用
19  attribute         属性访问（.）
20  attribute_expression  属性表达式的左侧
21  type              类型标注（最高，避免与 for..in 的 in 冲突）
```

### not/! 优先级修复

**问题**：之前 `not` 和 `!` 与一元运算符（`-`, `+`, `~`）同级（PREC.unary = 15），导致 `not a == b` 被解析为 `(not a) == b`，而非预期的 `not (a == b)`。

**修复**：将 `not`/`!` 的优先级设为 `PREC.boolean_not = 5`，高于 `and`(4) 但低于 `in`(6) 和 `compare`(7)。这样：

```gdscript
not a == b      # 解析为 not (a == b)  ✓
not a and b     # 解析为 (not a) and b  ✓
a and not b     # 解析为 a and (not b)  ✓
```

## Grammar 配置

### name & word

```js
name: "gdscript",
word: ($) => $._identifier,  // 用于关键字消歧
```

`word` 告诉 tree-sitter 哪些 token 是"词"，确保关键字（如 `if`）不会匹配为标识符的一部分。

### extras

```js
extras: ($) => [$.comment, /[\s\uFEFF\u2060\u200B]/, $.line_continuation],
```

可在任意位置出现的 token：注释、空白字符（包括 BOM、零宽空格）、行续行符（`\` + 换行）。

### externals（14 个）

由 scanner.c 处理的外部 token：

| Token | 用途 |
|-------|------|
| `_newline` | 换行符 |
| `_indent` | 缩进增加 |
| `_dedent` | 缩进减少 |
| `_string_start` | 字符串开始引号 |
| `_string_content` | 字符串内容 |
| `_string_end` | 字符串结束引号 |
| `_string_name_start` | 字符串名开始（`&"`） |
| `_node_path_start` | 节点路径开始（`^"` / `@"`） |
| `]` `)` `}` | 闭合括号（让扫描器知道在括号内，避免发出 DEDENT） |
| `,` | 逗号（用于 BODY_END 检测） |
| `_body_end` | 复合语句体的结束标记 |

### inline

```js
inline: ($) => [$._simple_statement, $._compound_statement],
```

内联规则：在生成的 AST 中不会产生中间节点，直接展开。

### supertypes

```js
supertypes: ($) => [
  $._compound_statement, $._pattern,
  $._expression, $._primary_expression,
  $._attribute_expression, $._parameters,
],
```

超类型在 SCM 查询中可被直接匹配，方便写出通用的模式规则。

## 规则详解

### 原子规则 (Atoms)

| 规则 | 匹配 | 说明 |
|------|------|------|
| `_identifier` | `/[a-zA-Z_][a-zA-Z_0-9]*/` | 内部标识符，被多处引用 |
| `identifier` | `$._identifier` | 通用标识符节点 |
| `name` | `$._identifier` | 语句中的命名符号（函数名、类名等） |
| `comment` | `# ...` | 单行注释 |
| `true/false/null` | 关键字 | 布尔和空值字面量 |
| `static_keyword` | `"static"` | 静态关键字 |
| `remote_keyword` | `remote/master/puppet/remotesync/mastersync/puppetsync` | 多人同步关键字 |

#### Region（自定义新增）

```js
region_start: token(prec(100, seq("#region", /([ \t]+[^\r\n]*)?/))),
region_end:   token(prec(100, seq("#endregion", optional(/[^\r\n]*/)))),
region:       seq($.region_start, repeat($._statement), $.region_end),
```

用 `prec(100)` 确保优先于普通 `comment` 规则匹配。`region` 作为 `_compound_statement` 的一种，可以被折叠。

#### 字符串类型

| 规则 | 语法 | 说明 |
|------|------|------|
| `string` | `"..."` / `'...'` | 普通字符串（含转义） |
| `string_name` | `&"..."` | 字符串名（本地化） |
| `node_path` | `^"..."` | 节点路径 |
| `get_node` | `$path` / `%path` | 获取节点快捷语法 |

字符串由 scanner.c 多段处理：`_string_start` → `_string_content` × N → `_string_end`。

#### 数字

| 规则 | 匹配 |
|------|------|
| `integer` | 十进制（含 `_` 分隔）、`0x` 十六进制、`0o` 八进制、`0b` 二进制 |
| `float` | `1.0`、`.5`、`1.`、`1e10`、`1.5e-3` 等变体 |

#### 转义序列

```
\uXXXX    4 位 Unicode
\UXXXXXX  6 位 Unicode
\xXX      2 位十六进制
\oOOO     3 位八进制
\r\n      回车换行
\其他     单字符转义（\n, \t 等）
```

---

### 类型 (Type)

```js
type: ($) => prec(PREC.type, choice($.attribute, $.identifier, $.subscript)),
```

使用最高优先级（21）避免与 `for_statement` 中的 `in` 关键字产生冲突。支持：

- 简单类型：`int`, `String`, `MyClass`
- 泛型/嵌套类型（通过 subscript）：`Array[int]`
- 属性路径类型：`Resource.Loader`

### 语句 (Statements)

#### body 规则

body 是 GDScript 语法的核心——Python 风格的缩进块：

```js
body: ($) => choice(
  $._simple_statements,                    // 单行体：if cond: pass
  $._newline,                              // 空体：if cond:\n（下一行是 dedent）
  $._body_end,                             // 括号/逗号前的结束
  seq($._indent, repeat($._statement),     // 缩进块
      choice($._body_end, $._dedent)),
),
```

四种变体覆盖了所有情况：单行、空行、括号内结束、标准缩进块。

#### 简单语句

```
_simple_statement:
  ├── annotations          # @export, @onready 等
  ├── signal_statement     # signal name(params)
  ├── class_name_statement # class_name Name, "icon" extends Parent
  ├── extends_statement    # extends Parent
  ├── expression_statement # 表达式、赋值、增强赋值
  ├── export_variable_statement  # export var / export onready var
  ├── onready_variable_statement # onready var
  ├── variable_statement   # var / static var / remote var
  ├── const_statement      # const
  ├── return_statement     # return expr
  ├── pass_statement       # pass
  ├── break_statement      # break
  ├── breakpoint_statement # breakpoint（调试用）
  └── continue_statement   # continue
```

简单语句可用分号分隔写在同一行，以换行或 body_end 结尾。

#### 复合语句

```
_compound_statement:
  ├── if_statement           # if/elif/else
  ├── for_statement          # for x:Type in expr:
  ├── while_statement        # while cond:
  ├── function_definition    # func name(params) -> Type:
  ├── constructor_definition # func _init(params):
  ├── class_definition       # class Name extends Parent:
  ├── enum_definition        # enum Name { A, B = 2 }
  ├── match_statement        # match expr:
  └── region                 # #region ... #endregion（自定义新增）
```

---

### 注解 (Annotations)

```js
annotation: prec.right(seq("@", $.identifier, optional(field("arguments", $.arguments)))),
_annotations: repeat1($.annotation),
```

注解如 `@export`, `@export_range(0, 100)` 等。`_annotations` 用 `repeat1` 允许多个注解堆叠，作为容器节点使语法树更清晰。

---

### 变量 (Variables)

变量声明的完整形式：

```gdscript
@export                    # 可选注解
static                     # 可选 static
var name                   # 变量名
  : Type                   # 可选类型标注
  = value                  # 可选初始值
  set(value): body get: body  # 可选 set/get
```

各种赋值变体：

| 变体 | 语法 |
|------|------|
| `_variable_assignment` | `= value` |
| `_variable_inferred_type_assignment` | `:= value` 或 `: = value` |
| `_variable_typed_assignment` | `: Type = value` |
| `_variable_typed_definition` | `: Type` 或 `: Type = value` |

### SetGet

GDScript 4.x 中属性的 setter/getter 有多种写法：

**内联形式**：
```gdscript
var prop = 10 set = _set_prop, get = _get_prop
```

**多行缩进形式**：
```gdscript
var prop:
    set(value):
        _prop = value
    get:
        return _prop
```

**旧式 setget 语法**（兼容 3.x）：
```gdscript
var prop = 10 setget _set_prop, _get_prop
```

`get` 的参数被别名化为 `()` （匿名节点），因为 getter 不接受参数。

### 控制流

#### if/elif/else

```js
if_statement: seq("if", condition, ":", body, repeat(elif_clause), optional(else_clause)),
```

支持链式 elif 和可选 else。

#### for

```gdscript
for x: int in range(10):  # 可选类型标注
    print(x)
```

类型标注用 `prec(PREC.type)` 避免与 `in` 关键字冲突。

#### while

```gdscript
while condition:
    body
```

---

### 类 (Class)

```js
class_definition: seq(optional(annotations), "class", name, optional(extends), ":", class_body),
```

`class_body` 与 `body` 的区别：类体只能包含类成员，不能包含裸表达式或控制流。

```
_class_member:
  ├── _simple_class_member  # const, extends, pass, signal, variable
  └── _compound_class_member  # class, enum, function, region
```

注意 `_compound_class_member` 中包含了 `region`（自定义新增），使得 `#region`/`#endregion` 可以在类体内使用且可折叠。

---

### 枚举 (Enum)

```gdscript
enum { A, B, C }               # 匿名枚举
enum Name { A, B = 5, C = B }  # 命名枚举
```

`_enumerator_expression` 限制等号右侧只能是简单表达式（整数、二元运算、标识符、一元运算、属性、下标、调用、括号表达式），不能是任意表达式。

---

### Match 语句

```gdscript
match value:
    pattern1:
        body1
    pattern2 when guard:
        body2
    var x, ..:
        body3
```

关键设计决策：

- `pattern_binding`（`var x`）和 `pattern_open_ending`（`..`）被注入到 `array` 和 `dictionary` 规则中，而非创建独立的 pattern 数组/字典规则。虽然这些只在 pattern 上下文中有意义，但简化了语法。
- `dictionary` 接受逗号分隔的混合键和键值对，也只在 pattern 中合法。
- 每个 `pattern_section` 只能有一个 `pattern_guard`（`when` 条件）。
- `match_body` 内部支持 `region_start`/`region_end`（自定义新增），实现 region 标记不影响 match 体解析。

### 表达式 (Expressions)

```
_expression:
  ├── _primary_expression  # 主要表达式
  └── conditional_expression  # 三元：x if cond else y

_primary_expression:
  ├── binary_operator       # 二元运算
  ├── identifier            # 标识符
  ├── string/integer/float/true/false/null  # 字面量
  ├── not_operator          # not x / !x
  ├── unary_operator        # -x / +x / ~x
  ├── string_name/node_path/get_node  # 特殊字符串
  ├── attribute             # a.b.c
  ├── subscript             # a[b]
  ├── base_call             # .method(args)
  ├── call                  # func(args)
  ├── array/dictionary      # [1,2] / {k:v}
  ├── parenthesized_expression  # (expr)
  └── await_expression      # await expr
```

#### _attribute_expression

```js
_attribute_expression: prec(PREC.attribute_expression, choice(
  $.identifier, $.string, $.integer, $.float, ...)),
```

这是属性链左侧允许的表达式类型。**排除了 `binary_operator` 和 `unary_operator`**，防止 `1 + a.b.c` 被错误解析为 `(1 + a.b).c`。

### 运算符

#### 二元运算符（24 种）

| 运算符 | 优先级 |
|--------|--------|
| `or` `||` | 3 |
| `and` `&&` | 4 |
| `not in` `in` | 6 |
| `<` `<=` `==` `!=` `>=` `>` | 7 |
| `\|` | 8 |
| `&` | 9 |
| `^` | 10 |
| `<<` `>>` | 11 |
| `+` `-` | 12 |
| `*` `/` `**` `%` | 13 |
| `as` | 17 |
| `is not` `is` | 16 |

#### not 运算符

```js
not_operator: prec(PREC.boolean_not, seq(choice("not", "!"), $._primary_expression)),
```

优先级 5，低于比较运算符(7)，高于 and(4)。

#### 一元运算符

```js
unary_operator: choice(
  prec(PREC.unary, seq("-", $._primary_expression)),  // -x
  prec(PREC.unary, seq("+", $._primary_expression)),  // +x
  prec(PREC.unary, seq("~", $._primary_expression)),  // ~x
),
```

优先级 15。

### 访问器 (Accessors)

#### attribute

```js
attribute: prec(PREC.attribute, seq(
  $._attribute_expression,
  repeat1(seq(".", choice($.attribute_subscript, $.attribute_call, $.identifier))),
)),
```

`repeat1` 处理链式点访问：`a.b.c.d`。

`attribute_call` 和 `attribute_subscript` 的存在是为了扁平化 AST——将 `obj.method(args)` 解析为扁平的 `call` 节点而非嵌套结构。

#### subscript

```gdscript
arr[0]
dict["key"]
```

使用 `PREC.attribute`(19) 高优先级解决歧义，例如 `class C: var x = arr[0]`。

---

### 条件表达式

```gdscript
value if condition else other_value
```

使用 `prec.right` 右结合，优先级 -1（最低）。

---

### 赋值

```gdscript
x = value                    # 简单赋值
x += value                   # 增强赋值（+=, -=, *=, /=, **=, %=, >>=, <<=, &=, ^=, |=）
```

赋值不属于表达式，而是独立的 `expression_statement` 选择项。

### 数据结构

#### Array

```gdscript
[1, 2, 3]
[var x, ..]   # pattern 中使用
```

#### Dictionary

```gdscript
{"key": value}
{key = value}           # 标识符键用 = 连接
{"key"}                 # pattern 中仅匹配键
{"key": var x, ..}      # pattern 绑定和开放结尾
```

#### Pair

```gdscript
key: value      # 通用键值对（冒号分隔）
key = value     # 标识符键值对（等号分隔）
key: var x      # pattern 绑定
```

### 函数 (Functions)

#### 函数定义

```gdscript
@export
static func name(param1: int = 0, param2: String) -> bool:
    return true
```

完整的函数定义支持：可选注解、可选 static/remote、可选函数名、参数、返回类型、body。

**抽象函数**：函数可以没有 body，以 `;` 或换行结尾：
```gdscript
func abstract_method();  # 无 body
```

#### 构造函数

```gdscript
func _init(params).super_args(args) -> void:
    body
```

支持 `.super_args(args)` 调用父类构造函数。

#### Lambda

```gdscript
var fn = func(x): return x * 2
var named = func multiply(x, y): return x * y
```

Lambda 存在于 `_rhs_expression` 中（不在 `_expression` 中），只能出现在赋值右侧、函数参数等位置。

#### 参数类型

| 类型 | 语法 |
|------|------|
| 简单参数 | `name` |
| 类型参数 | `name: Type` |
| 默认参数 | `name = value` |
| 类型默认参数 | `name: Type = value` 或 `name := value` |
| 可变参数 | `...params` |

`typed_parameter` 使用 `PREC.typed_parameter`(-1)，避免在参数列表中产生歧义。

---

### 函数调用

```gdscript
func(args)           # call
obj.method(args)     # call + attribute
.method(args)        # base_call（super 调用模式）
```

`base_call`（`.method()`）用于在子类中调用父类方法。

---

### Await

```gdscript
await signal_name
await coroutine_func()
```

### 辅助函数

```js
sep1(rule, sep)           // rule (sep rule)*
trailSep1(rule, sep)      // sep1 + optional(sep)
commaSep1(rule)           // sep1(rule, ",")
trailCommaSep1(rule)      // trailSep1(rule, ",")
```

`trailCommaSep1` 允许末尾逗号，GDScript 中广泛使用。
