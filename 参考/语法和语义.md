# 语法高亮 vs 语义高亮

## 疑问

类的属性在类成员函数中使用时，没有按照 property 高亮着色。

## 原因

这是 tree-sitter 的固有局限 — 它只能做**语法高亮**，无法做语义分析。

当前属性高亮规则只匹配声明处：

```scheme
(class_definition
  (class_body
    (variable_statement
      (name) @property)))
```

示例：

```gdscript
class MyClass:
    var my_prop = 10        # 声明 → 匹配 → @property

    func my_method():
        print(my_prop)       # 使用 → 只是 (identifier) → @variable
```

`my_prop` 在方法体内使用时是一个普通的 `(identifier)` 节点，tree-sitter 无法区分它引用的是类属性还是局部变量 — 这需要语义分析，超出了 tree-sitter 的能力范围。

## 可行的替代方案

通过 `self.` 访问时可以正确高亮：

```gdscript
func my_method():
    print(self.my_prop)      # my_prop → @property
```

因为已有规则匹配 `attribute` 节点中的 `identifier` 为 `@property`：

```scheme
(attribute
  (_)
  (identifier) @property)
```
